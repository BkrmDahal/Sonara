# Contributing to Sonara

Thank you for your interest in contributing to Sonara! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone <your-fork-url>`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages
7. Push and create a Pull Request

## Code Style

- Use consistent indentation (2 spaces)
- Follow existing code patterns
- Add comments for complex logic
- Keep functions focused and small
- Use meaningful variable names

## Testing

Before submitting:
- Test in Chrome (latest version)
- Test with and without OpenAI API key
- Test on different websites
- Check browser console for errors
- Verify all features work as expected

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add dark mode support`
- `fix: Resolve audio generation timeout`
- `docs: Update README with new features`
- `refactor: Clean up storage utilities`

## Pull Request Process

1. Update README.md if needed
2. Add tests if applicable
3. Ensure all checks pass
4. Request review from maintainers
5. Address feedback promptly

## Areas for Contribution

- Bug fixes
- Performance improvements
- New features (check roadmap)
- Documentation improvements
- UI/UX enhancements
- Accessibility improvements

## Questions?

Open an issue for discussion before starting work on major features.
